🔐 Research Project: LLM-Assisted Security Analysis

Using Large Language Models to Support Security Review in Single-Page Web Applications

A research project exploring how Large Language Models can improve vulnerability detection, reduce false positives, and speed up security review in modern React and Node.js applications.

About the research

Modern single-page web applications push more logic into the browser, which creates new security risks at the intersection of frontend behavior, backend APIs, token handling, and access control. This work examines whether an LLM can serve as a practical assistant during security review.

Research question

Can a structured LLM-assisted review workflow help developers identify vulnerabilities in single-page web applications more effectively than a lightweight developer-led baseline?

Targeted vulnerability categories

  • Cross-site scripting through unsafe content rendering
  • Insecure JWT storage in browser localStorage
  • Missing server-side authorization checks

Methodology

The workflow uses an LLM as an analytical aid, not as a replacement for engineering judgment. Findings are reviewed and validated by a developer before being accepted.

1

Extract

Application code modules are segmented by role and component.

2

Preprocess

Each segment is prepared with context about its purpose in the system.

3

Prompt

Structured instructions guide the LLM toward specific security review tasks.

4

Analyze

The model returns findings, severity notes, code evidence, and remediation ideas.

5

Validate

A developer verifies which findings are correct and records the outcome.

Results

Across ten structured test scenarios, the LLM-assisted review produced higher detection coverage, fewer false positives, and significantly faster review times than the baseline process.

Notable findings

  • The LLM consistently identified unsafe rendering patterns that create stored XSS risks.
  • It connected localStorage-based JWT storage to the broader impact of XSS more effectively than the baseline review.
  • It identified authorization weaknesses hidden behind correct-looking frontend restrictions.

Why this matters

The strongest value of the LLM-assisted workflow is not only detection. It combines identification, explanation, and remediation guidance in a way that can shorten the path from issue discovery to developer action.
